# Bilibili App 接口模拟 - v8.62.0

本项目逆向分析了哔哩哔哩（Bilibili）安卓客户端 v8.62.0 版本，并使用 Python 实现了与播放量上报相关的两个核心接口（https://api.bilibili.com/x/report/click/android2  、"https://api.bilibili.com/x/report/heartbeat/mobile"）的模拟访问。

## 工作原理

本项目的核心是模拟未登陆游客用户点击视频来增长播放量。通过**模拟生成**一套完整的、一致的安卓设备指纹来与Bilibili服务器进行交互，该指纹包括但不限于：

*   `buvid` (b站自设设备ID)
*   `oaid` (开放匿名设备标识符)
*   `x-bili-ticket`（JWT)
*   设备型号、系统版本等静态参数

**所有这些设备信息均为随机生成或硬编码的伪造数据，不涉及也绝不会读取或暴露您真实的设备信息。**

## 用前须知

- 本项目完全还原了两个接口的所有参数而不只是必需参数（包括JWT)，理论上能极大程度的降低风控概率；
- 但b站目前对游客的播放风控很严，通过这种方式模拟的请求每分钟增长的播放量在个位数；
- 而且游客能增长的播放量有上限，不能一直刷到上万播放量的；
- 暂时没有考虑添加IP代理池，因为根据一些之前类似项目的经验加了代理池播放量增速也是个位数

## 使用方式

将本项目克隆到本地，直接运行**click.py**或者修改**click.py**文件里的main函数，填入目标视频的BV号和想要模拟观看的时长（一般就是该视频的总时长，即完整播放），随后运行**click.py**即可。

可修改模拟请求的次数和等待间隔，默认100次、2-5秒
